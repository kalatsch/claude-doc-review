claude-doc-review

Turn any markdown document Claude writes into a human-friendly review page — clear prose, glossary tooltips, collapsible technical detail — with an inline text-selection commenting layer that syncs live to Claude. Everything runs on your machine; no server, no telemetry.

/doc-review inline comments & replies glossary tooltips collapsible detail light / dark 100% local

When Claude writes a plan or spec, it writes it for itself — dense, full of acronyms and shorthand. Reviewing that in a chat scrollback is painful. /doc-review rewrites the document into plain prose, explains every term, folds the deep detail away, and lets you select any sentence and comment — which Claude reads and answers in place. The doc becomes a back-and-forth review surface instead of a wall of text.

What you get

Humanized rewrite

Claude rephrases the terse source into clear prose, expanding jargon and acronyms on first use.

Glossary tooltips

A Словарь / glossary section becomes hover tooltips on every term — case-insensitive, inflection-aware.

Collapsible detail

Deep asides — data models, edge cases — fold into expandable blocks so the main read stays light.

Inline comments

Select any text to drop a thread anchored to that quote — an ongoing back-and-forth, not one-shot. Resolve when settled, reopen to continue.

Live sync with Claude

Comments round-trip through a local comments.json; Claude answers every open thread with a new message from you, live. Resolved threads are left alone.

Navigate & track

TOC with scroll-spy, next/prev comment jumps, open/total counts, light & dark — all 100% local.

How it works
  1. Run /doc-review path/to/doc.md on any markdown Claude wrote — a plan, a spec, a design note.
  2. Claude rewrites it into clear prose with a glossary and collapsible detail, and lays the page down in .claude/doc-review/<slug>/ (kept out of your project's git automatically).
  3. A tiny dependency-free local server starts and the page opens in your browser — read it, hover the glossary terms, expand the detail blocks.
  4. Select text → comment. Ask Claude to check; it answers every open thread and the page updates live — keep replying as many rounds as you need, it isn't one-shot. Resolve a thread when it's settled (Claude then leaves it alone); reopen it to continue.
Install
▸

Install from this repo

no publishing

Inside Claude Code — no central publishing involved. These are two separate slash commands: run them one at a time, in order.

1Add this GitHub repo as a plugin source
/plugin marketplace add kalatsch/claude-doc-review
2Install the plugin from it
/plugin install claude-doc-review@claude-doc-review
